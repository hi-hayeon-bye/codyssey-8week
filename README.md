## 설계 구조 (UI Hierarchy)
이 프로그램이 어떻게 구성되어 있는지 한눈에 볼 수 있도록 계층 구조를 설계했습니다.

#### 1. `QtWidgets`: UI의 물리적 구성 요소 (위젯)
사용자와 직접 상호작용하는 물리적인 부품들을 불러왔습니다.
* **`QApplication`**: 이 응용 프로그램이 '응용 프로그램'으로서 작동하게 만듭니다. 클릭 이벤트나 화면 업데이트를 총괄합니다.
* **`QWidget`**: 모든 UI 구성 요소의 조상 입니다. 우리는 이 `QWidget`을 상속받아 우리만의 '계산기 창'이라는 새로운 위젯을 설계했습니다.
* **`QVBoxLayout` & `QGridLayout`**: 이들은 부품이 아니라 '정렬 틀'입니다. 전체적인 **수직 구조**(`QVBoxLayout`)와 버튼들의 **격자 배치**(`QGridLayout`)를 위해 선택했습니다.
* **`QPushButton`**: 말 그대로 **버튼**입니다. 계산기의 0~9, +, - 등을 만들기 위해 필수적입니다.
* **`QLineEdit`**: 텍스트를 입력하거나 보여주는 **한 줄창**입니다. 계산기에서는 숫자가 출력되는 '결과 화면'으로 활용했습니다.

#### 2. `QtCore`: 엔진과 규칙 (코어)
눈에 보이지 않는 논리적인 규칙과 명령의 세부 옵션을 설정합니다.
* **`Qt`**: PyQt 안에서 쓰이는 수많은 설정값(상수)들의 모음집입니다. 예: `Qt.AlignmentFlag.AlignRight` (결과창 글자를 오른쪽으로 정렬해라) 같은 **디테일한 명령**을 주기 위해 필요했습니다.

#### 3. `QtGui`: 디자인과 감성 (그래픽)
사용자에게 보여지는 시각적인 스타일을 담당합니다.
* **`QFont`**: 계산기 숫자가 너무 작으면 읽기 힘들겠죠? 숫자의 **글꼴과 크기**를 조절해서 가독성을 높이기 위해 가져왔습니다.

## 🛠️ 고민들... (Deep Thinking)

### 1. "레이아웃을 어떻게 짤까?"에 대한 고민
**수직(Vertical) 안에 격자(Grid)를 넣는 방식**을 선택했습니다. 
* **이유**: 결과창은 무조건 위에 고정되어야 하고, 버튼들은 바둑판처럼 정렬되어야 하기 때문입니다.
* 특히 **'0' 버튼**은 혼자 두 칸을 차지해야 해서, `QGridLayout`의 `columnSpan` 기능을 사용해 정밀하게 배치했습니다.

### 2. "효율적인 코드란 무엇일까?"
버튼이 20개나 되는데 함수를 20개 만드는 건 너무 비효율적이라 생각했습니다. 그래서 `self.sender()`를 사용해 **어떤 버튼을 눌러도 하나의 함수(`press_event`)에서 처리**하도록 만들었습니다. 덕분에 코드가 깔끔해졌고 전공자다운 설계가 되었습니다.

### 3. "에러가 나면 어떻게 하지?"
화성에서 계산기가 꺼지면 생존에 직결됩니다. `5 / 0` 같은 잘못된 수식을 넣어도 프로그램이 죽지 않고 **'Error'라고 알려주도록 `try-except` 예외 처리**를 꼼꼼하게 넣었습니다.
